# AI Context Installer — Design & Parity Notes

## What It Does

The `tina4python ai` command installs AI coding assistant context files into the user's project.
Each of the 7 supported tools gets a **tool-specific file** with content tailored to that tool's
format and conventions — not a single generic blob.

## Supported Tools

| # | Tool | File | Content Style |
|---|------|------|---------------|
| 1 | Claude Code | `CLAUDE.md` | Full developer guide (500+ lines), copied from framework CLAUDE.md |
| 2 | Cursor | `.cursorules` | Compact key patterns + rules (~45 lines) |
| 3 | GitHub Copilot | `.github/copilot-instructions.md` | Short instructions + rules (~30 lines) |
| 4 | Windsurf | `.windsurfrules` | Key patterns + structure (~60 lines) |
| 5 | Aider | `CONVENTIONS.md` | Conventions + patterns (~58 lines) |
| 6 | Cline | `.clinerules` | Key patterns + conventions (~42 lines) |
| 7 | OpenAI Codex | `AGENTS.md` | Task-oriented agent instructions (~70 lines) |

## How It Works

1. `generate_context(tool_name)` — returns tool-specific Markdown content
2. `_install_for_tool()` — writes the content to the correct file path
3. For Claude Code, also copies `.claude/skills/` and `.claude/commands/` from the framework

## Content Structure (per tool)

All tools receive:
- Framework name and version (auto-read from `__version__`)
- Route pattern example
- ORM pattern example
- Key conventions (7 rules)
- Built-in feature list (compact one-liner)

Longer tools also get:
- Project structure
- CLI commands
- Additional code examples

## Cross-Framework Parity

This exact same architecture must exist in all 4 backends:
- **Python**: `tina4_python/ai/__init__.py` — `generate_context(tool_name)`
- **PHP**: `Tina4/AI.php` — `AI::generateContext($toolName)`
- **Ruby**: `lib/tina4/ai.rb` — `Tina4::AI.generate_context(tool_name)`
- **Node.js**: `packages/core/src/ai.ts` — `generateContext(toolName)`

Each framework's content must be language-specific (Python examples in Python, PHP in PHP, etc.)
but follow the same structural pattern.

## Version Handling

The version is read dynamically from the framework's version constant at generation time —
never hardcoded in the generator. This ensures `tina4python ai` always produces context
matching the installed version.

## Repo-Level AI Files

Each Tina4 repo also ships pre-generated AI context files that are committed to Git:
- `CLAUDE.md` — maintained manually (the authoritative developer guide)
- `.cursorrules` — committed, matches `generate_context("cursor")` output
- `.github/copilot-instructions.md` — committed
- `llms.txt` — committed (reference doc, not generated by installer)
- `.windsurfrules`, `.clinerules`, `CONVENTIONS.md`, `AGENTS.md` — generated on demand
