# Internationalization (I18n)

The `I18n` class provides translation support using JSON locale files. It supports dot-notation keys, parameter substitution with `{placeholder}` syntax, locale fallback chains, and runtime locale switching. Configuration is driven by environment variables.

## Locale File Format

Locale files are JSON files named by locale code, stored in `src/locales/` by default.

```json
// src/locales/en.json
{
    "greeting": "Hello, {name}!",
    "nav": {
        "home": "Home",
        "about": "About Us",
        "contact": "Contact"
    },
    "errors": {
        "not_found": "Page not found",
        "forbidden": "You do not have access to this page",
        "server_error": "Something went wrong"
    },
    "items_count": "You have {count} items in your cart"
}
```

```json
// src/locales/fr.json
{
    "greeting": "Bonjour, {name} !",
    "nav": {
        "home": "Accueil",
        "about": "A propos",
        "contact": "Contact"
    },
    "errors": {
        "not_found": "Page non trouvee",
        "forbidden": "Vous n'avez pas acces a cette page"
    },
    "items_count": "Vous avez {count} articles dans votre panier"
}
```

## Basic Usage

```php
use Tina4\I18n;

$i18n = new I18n('src/locales', defaultLocale: 'en');

// Simple translation
echo $i18n->translate('nav.home');     // "Home"
echo $i18n->t('nav.about');           // "About Us" (shorthand alias)

// With parameter substitution
echo $i18n->t('greeting', ['name' => 'Alice']);
// "Hello, Alice!"

echo $i18n->t('items_count', ['count' => 3]);
// "You have 3 items in your cart"

// Nested keys (dot notation)
echo $i18n->t('errors.not_found');    // "Page not found"
echo $i18n->t('errors.forbidden');    // "You do not have access to this page"
```

## Switching Locales

```php
$i18n->setLocale('fr');
echo $i18n->getLocale(); // "fr"

echo $i18n->t('greeting', ['name' => 'Alice']);
// "Bonjour, Alice !"

echo $i18n->t('nav.home');
// "Accueil"
```

## Locale Fallback

If a key is missing in the current locale, the default locale is used. If it is also missing there, the key itself is returned.

```php
$i18n->setLocale('fr');

// Missing in French, falls back to English
echo $i18n->t('errors.server_error');
// "Something went wrong"

// Missing in both locales, returns the key
echo $i18n->t('nonexistent.key');
// "nonexistent.key"
```

## Per-Call Locale Override

```php
// Translate in a specific locale without changing the active one
echo $i18n->t('nav.home', locale: 'fr');
// "Accueil"

echo $i18n->getLocale(); // Still "en"
```

## Available Locales

```php
$locales = $i18n->getAvailableLocales();
// ['en', 'fr', 'de', 'es'] — based on JSON files found
```

## Adding Translations at Runtime

```php
$i18n->addTranslation('en', 'custom.key', 'Custom value');
echo $i18n->t('custom.key'); // "Custom value"
```

## Loading All Translations

```php
$translations = $i18n->loadTranslations('en');
// ['greeting' => 'Hello, {name}!', 'nav.home' => 'Home', ...]
```

## Environment Variables

| Variable | Default | Description |
|----------|---------|-------------|
| `TINA4_LOCALE` | `en` | Default locale code |
| `TINA4_LOCALE_DIR` | `src/locales` | Directory containing locale JSON files |

## Using in Templates

```php
use Tina4\Frond;

$frond = new Frond('src/templates');

// Register a translation filter or global
$frond->addGlobal('t', function (string $key, array $params = []) use ($i18n) {
    return $i18n->t($key, $params);
});
```

```html
<h1>{{ t('greeting', {'name': user.name}) }}</h1>
<nav>
    <a href="/">{{ t('nav.home') }}</a>
    <a href="/about">{{ t('nav.about') }}</a>
</nav>
```

## Using in Routes

```php
use Tina4\Router;
use Tina4\Request;
use Tina4\Response;

$i18n = new I18n();

Router::get('/api/greeting/{name}', function (Request $request, Response $response) use ($i18n) {
    $locale = $request->queryParam('lang', 'en');
    $name = $request->params['name'];

    $greeting = $i18n->t('greeting', ['name' => $name], locale: $locale);

    return $response->json(['message' => $greeting]);
});

// GET /api/greeting/Alice?lang=fr
// {"message": "Bonjour, Alice !"}
```

## Tips

- Use dot notation for organizing translations hierarchically (`nav.home`, `errors.not_found`).
- Nested JSON objects are automatically flattened with dot separators during loading.
- Keep locale files small and focused — split by feature if needed.
- The `{placeholder}` syntax is simple string replacement, not an expression parser.
- Missing translations return the key name, making untranslated strings easy to spot.
- Use `getAvailableLocales()` to build a language selector UI.
